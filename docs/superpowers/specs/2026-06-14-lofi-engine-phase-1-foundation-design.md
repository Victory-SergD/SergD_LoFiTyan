# Фаза 1 «Фундамент» — дизайн

**Проект:** Lofi Engine (форк meel-hd/lofi-engine, Tauri 2 + Svelte 3 + TypeScript + Tone.js)
**Дата:** 2026-06-14
**Статус:** утверждён пользователем (подход B, дизайн в целом — ok)

## 1. Контекст и цель

Это первая из четырёх фаз большого видения (полный список — в разделе «Дорожная карта»). Фаза 1 — **фундамент**: сделать продукт стабильным и приятным до того, как строить новые фишки.

**Цель Фазы 1:**
- Починить все 30 подтверждённых багов из аудита кода.
- Сделать так, чтобы приложение полноценно работало на **вертикальном (портретном) мониторе** — раскладка «вариант B»: большой холст сверху, компактная панель-плеер снизу.
- **Просторный режим**: управление само скрывается при простое мыши, освобождая холст под будущие сцены/видео/персонажа.
- Честный UX «треков»: то, что называется «треками», — это слои атмосферы; переименовать в **«Атмосфера»**, а смену музыки вынести в явную кнопку 🔄 «Новая музыка».

**Принцип:** упор на качество. На сломанном фундаменте новые режимы (API-станции, видео, персонаж) строить нельзя.

## 2. Рамки

**В рамках Фазы 1:**
- 30 исправлений багов (см. раздел 6).
- Адаптивная вёрстка + портретный режим (вариант B).
- Просторный режим (авто-скрытие управления + ручной тумблер и хоткей).
- Целевой рефактор звуковой/трековой подсистемы в Svelte-сторы.
- Переименование «Tracks» → «Атмосфера», кнопка 🔄 «Новая музыка».
- Слой «Холст» как отдельный компонент, архитектурно готовый принять видео и персонажа.
- Тесты на чистую логику + ручной QA-чеклист по всем кнопкам.

**Вне рамок (последующие фазы):**
- Фаза 2 — «Станции»: второй режим музыки через открытые lofi-API, переключатель Генеративный ↔ Стриминг, автоопределение доступности источников.
- Фаза 3 — «Вертикальный холст»: реальные движущиеся сцены, картинки, мини-видео.
- Фаза 4 — «Персонаж»: интерактивный LoFi-тян, перемещается по экрану, реагирует, позже — голосовой заказ музыки.

Холст-слой из Фазы 1 проектируется так, чтобы Фазы 3–4 встраивались в него без переделки раскладки.

## 3. Утверждённый подход

**Подход B — целевой рефактор** (выбран пользователем) вместо точечных заплаток.

Звуковая/трековая часть сегодня разбросана по трём независимым путям (TrackList/index.svelte, TrackListItem.svelte, клавиатурный обработчик 1–9) + дублирующееся состояние громкостей и режима Авто-DJ. Это корень ~8 багов (задвоение звука, рассинхрон подсветки, утечки слушателей).

Решение: ввести **единый источник правды** — Svelte-сторы:
- `atmosphereStore` — список слоёв атмосферы, их состояние play/stop и громкость; единственный владелец `Audio`-элементов.
- `volumeStore` — громкости (главная + по эффектам), вместо `localStorage`-поллинга.
- `autoDjStore` — режим Авто-DJ (один экземпляр вместо трёх).
- `engineStore`/события — текущая генеративная музыка и команда «Новая музыка» (регенерация прогрессии).

Остальные баги (вёрстка, окно, бэкенд, i18n) чинятся точечно по списку аудита.

## 4. Архитектура

### 4.1 Слой «Холст» (новый компонент)
`Canvas` — отдельный слой во всю область приложения: фон (bg1–10.webp) + визуализатор. В Фазе 1 показывает текущие фоны; имеет чёткий интерфейс для будущих слоёв контента (видео, персонаж) — например, slot/проп `children` и z-упорядочивание, чтобы Фазы 3–4 добавляли контент, не трогая раскладку.

### 4.2 Раскладка и адаптивность (вариант B)
- Окно (`tauri.conf.json`): `resizable: true`, `minWidth: 360`, `minHeight: 560`, портретно-дружелюбные дефолты (~480×900). Без `maxWidth/maxHeight`.
- Включить **уже написанную, но мёртвую** мобильную вёрстку: поднять брейкпоинт (сейчас `max-width:600px` при `minWidth:930` — никогда не срабатывает) и/или добавить `@media (orientation: portrait)`.
- В портрете корневой контейнер переключается с `flex-direction: row` на `column`: сверху `Canvas`, снизу док-панель (плеер + атмосфера + настройки).
- В портрете снять `overflow:hidden` с `html/body/.container` → `overflow-y:auto` (контент прокручивается, не обрезается).
- Визуализатор и панели (TrackList/Info/Settings), завязанные на vw/vh под ландшафт, получают портретные переопределения (через `clamp()`/`min()` и orientation-медиазапросы).
- Ландшафтный режим сохраняется рабочим — это тот же responsive-слой.

### 4.3 Просторный режим (авто-скрытие управления)
- `immersionStore` (или util) отслеживает активность мыши/клавиатуры. Нет событий ~3 сек → класс `.immersive` на корне → плавное затухание «хрома» (полоса + док), холст разворачивается.
- `mousemove`/нажатие клавиши → таймер сбрасывается, управление возвращается.
- Ручной тумблер «Просторный режим» (кнопка) + горячая клавиша.
- Все слушатели регистрируются в `onMount` с очисткой в `onDestroy`.

### 4.4 Звук/Атмосфера (рефактор, см. 3)
Единые сторы; компоненты (TrackList, TrackListItem, клавиатура, Авто-DJ WORLD) только диспатчат действия в стор, который владеет аудио и состоянием. Никаких мутаций пропов на месте и `push()` без переприсваивания.

### 4.5 Бэкенд (Rust/Tauri)
- Зарегистрировать `tauri_plugin_shell` в `lib.rs` (`.plugin(tauri_plugin_shell::init())`), внешние ссылки открывать через `@tauri-apps/plugin-shell` `open(url)` (заработает ссылка на GitHub). Альтернатива — удалить мёртвую зависимость; **решение: подключить**, ссылки нужны.
- Включить CSP в `tauri.conf.json` (вместо `null`).
- Грейсфул-обработка ошибки старта вместо `.expect()`/panic.

## 5. Новые/изменённые элементы UX

| Элемент | Было | Стало |
|---|---|---|
| «Tracks» | вводит в заблуждение (это не песни) | «**Атмосфера**» — слои звуков природы |
| Смена музыки | неочевидно (кнопка refresh) | явная кнопка 🔄 «**Новая музыка**» в плеере |
| Хром (полоса/панель) | всегда на экране | **авто-скрытие** при простое + ручной «Просторный режим» |
| Кнопки эффектов | подсветка врёт о состоянии | подсветка только после реального старта звука |
| Кнопки окна / пробел | работают непредсказуемо | единая логика, реагируют стабильно |

## 6. Полный список исправлений (30 багов аудита)

**Блок 1 — Вертикальный режим / холст**
- `layout-1` окно: relax min-dimensions, `resizable:true`, портретные дефолты.
- `layout-2` корневой контент `flex-direction:row` → reflow в `column` в портрете.
- `layout-3` брейкпоинт 600px при minWidth 930 не срабатывает → поднять брейкпоинт / orientation-запросы.
- `layout-5` визуализатор фикс-180px/absolute → `clamp()`, портретные правила.
- `layout-7` TrackList/Info/Settings в vw/vh под ландшафт → портретные переопределения.
- `layout-8` глобальный `overflow:hidden` обрезает → `overflow-y:auto` в портрете.

**Блок 2 — Звук и Атмосфера (рефактор)**
- `audio-1` (юзер-репорт) смена «трека» не трогает музыку → честный UX: переименование + кнопка 🔄; атмосфера и музыка разведены явно.
- `audio-2` `Tone.start()` без await → `await` перед запуском транспорта/секвенсеров.
- `audio-3` `activeAudios` мутируется на месте, задвоение → стор + переприсваивание + guard.
- `audio-4` `TrackListItem.pauseTrack` не удаляет из `activeAudios` → логика в сторе родителя.
- `audio-5` `setInterval` громкости не очищается → стор громкости, без поллинга.
- `audio-7` WORLD Авто-DJ десинхронит треки → через единый стор.
- `state-2` `TrackListItem` мутирует проп `track.isPlaying` → события/колбэки в родителя.
- `state-4` слушатели/таймеры не снимаются (утечки, дубли) → `onMount`+cleanup; один keydown на цифры.

**Блок 3 — Кнопки и UX-проход**
- `ui-1` (юзер-репорт) пробел в обход проверок → пробел вызывает `handleButtonAction()` как кнопка.
- `ui-2` (юзер-репорт) кнопки окна мертвы до загрузки appWindow → await импорта / `ready`-флаг.
- `ui-3` (юзер-репорт) подсветка эффектов врёт → флаг после `play().then()`.
- `ui-4` буквенные хоткеи срабатывают при вводе → guard на input/modifier.
- `ui-5` настройки моргают при mount → не дёргать видимое состояние при инициализации.
- `ui-6` «клик-вне» вешается повторно → `onMount`+cleanup.
- `ui-7` mac-«развернуть» не отражает состояние → читать `isMaximized()`.
- `ui-8` кнопка-«глаз» без обратной связи → настоящий тумблер с индикацией.

**Блок 4 — Бэкенд / безопасность / i18n**
- `backend-1` shell-плагин не зарегистрирован → зарегистрировать.
- `backend-2` panic на старте → грейсфул-обработка.
- `backend-3` CSP отключён → задать CSP.
- `backend-4` внешние ссылки не открываются → через shell `open()`.
- `backend-5` неиспользуемая зависимость shell → используется после backend-1/4.
- `backend-6` macOSPrivateApi (App Store) → задокументировать ограничение (без изменений кода).
  > **Примечание о распространении (backend-6):** `macOSPrivateApi: true` в `tauri.conf.json` нужен для прозрачного окна без рамки. Это приватный API Apple, поэтому **сборка не пройдёт в Mac App Store**. Прямое распространение (`.dmg` / прямое скачивание) — допустимо и является целевым каналом Фазы 1. Флаг НЕ меняем; при необходимости публикации в App Store его придётся отключить и вернуть стандартную рамку окна.
- `state-6` направление текста хардкод `ltr` → вычислять из локали.
- `state-7` Авто-DJ мигает дефолтом + 3 копии → инициализация из storage + общий стор.

## 7. Тестирование и приёмка

**Автотесты (на чистую логику):**
- Генерация прогрессии аккордов (детерминированность при seed, корректные переходы).
- Редьюсеры сторов: `atmosphereStore` (play/stop/двойной клик не задваивает), `volumeStore`, `autoDjStore`.

**Ручной QA-чеклист (проход «UX-дизайнера»):**
- Каждая кнопка: клик делает ровно то, что подписано; подсветка соответствует реальному состоянию.
- Пробел и экранный play эквивалентны; звук стартует с первого раза.
- Кнопки окна (закрыть/свернуть/развернуть) реагируют с первого клика.
- Портрет: раскладка B корректна, ничего не обрезано, прокрутка работает; просторный режим прячет/возвращает управление.
- Ландшафт: всё по-прежнему работает.

**Финальная проверка:** `pnpm tauri:d`, прогон чеклиста в обеих ориентациях, проверка на реальном вертикальном мониторе пользователя.

**Критерии приёмки:** все 30 пунктов раздела 6 закрыты; чеклист зелёный; портретный и просторный режимы работают; сборка `pnpm tauri:b` проходит.
